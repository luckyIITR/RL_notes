Introduction to Reinforcement Learning

Lecture notes · March 2026

🧠 What is Reinforcement Learning?

  • A major pillar of machine learning alongside supervised and unsupervised learning.
  • Not as widely deployed commercially yet, but has strong and growing research support.
  • Core idea: instead of telling the algorithm the correct answer for every input, you specify a reward function that signals when it is doing well or poorly.
  • The algorithm's job is to automatically figure out how to take good actions to maximise rewards over time.
Key insight: You tell the system what to achieve, not how to achieve it. This flexibility is what makes RL powerful.

🚁 Motivating Example — Autonomous Helicopter

  • Stanford autonomous helicopter: 32 lbs, equipped with onboard computer, GPS, accelerometers, gyroscopes, and a magnetic compass.
  • Task: 10 times per second, given current position/orientation/speed, decide how to move the two control sticks.
  • Radio-controlled helicopters are significantly harder to keep balanced than quadrotors.
  • Using RL, the helicopter learned to perform aerobatic maneuvers — including flying upside down.

📖 Core Terminology

Term Definition Helicopter Example
State (s) All information describing the current situation of the agent. Position, orientation, speed of the helicopter.
Action (a) A decision/move taken by the agent in a given state. How far to push each control stick.
Reward A scalar signal indicating how good or bad the action/outcome was. +1/sec when flying well; −1000 on crash.
Policy (π) A function mapping states to actions — what the agent has learned. The trained mapping s → a that keeps the helicopter airborne.

⚖️ Why Not Supervised Learning?

  • You could in principle collect states (x) and ask an expert pilot for the best action (y), then train a neural network with supervised learning.
  • Problem: In continuous control tasks, the "correct" action is highly ambiguous — tilt a little left? A lot left? Increase throttle a little or a lot?
  • It is very difficult to collect a clean dataset of ideal (state → action) pairs for robotics tasks.
  • Reinforcement learning avoids this by only requiring a reward signal, not labelled optimal actions.

🏆 The Reward Function

  • Analogous to training a dog: you can't demonstrate every desired behaviour, so you reward good behaviour and penalise bad behaviour.
  • The RL algorithm figures out, on its own, how to get more rewards and avoid penalties.

  • RL is less widely deployed than supervised learning today, but is a critical and fast-growing area of ML research.
  • You do not need to specify the correct action for every state — only a reward signal.
  • The power of RL is in its generality: specify what success looks like, and the agent discovers how to achieve it.
  • Next steps: formalise the RL problem mathematically and develop algorithms for automatically selecting good actions.

Lecture 3 — The Return & Discount Factor

💰 What is the Return?

  • The Return answers: "is this sequence of rewards better or worse than another?" — it gives a single number to compare action sequences.
  • Analogy: A $5 bill at your feet vs. a $10 bill half an hour away. $10 > $5, but the effort (time) makes the closer reward more attractive. The return captures this preference for sooner rewards.
  • Formally, the return is the discounted sum of all future rewards from the current time step until the terminal state.
General Formula
Return = R1 + γ·R2 + γ²·R3 + γ³·R4 + …
where γ (gamma) is the discount factor, a number slightly less than 1 (e.g. 0.9, 0.99, 0.999).

⏳ The Discount Factor (γ)

  • γ is a number in (0, 1). Each additional time step multiplies the reward by another factor of γ, so rewards further in the future are worth less.
  • This makes the RL algorithm impatient — it prefers collecting rewards sooner rather than later.
  • R₁ gets full credit (×1), R₂ gets ×γ, R₃ gets ×γ², and so on.
  • Common practical values: γ = 0.99 or γ = 0.999 (mild discounting). For illustration, γ = 0.5 is used here (heavy discounting).
γ = 0.9 (mild)
0 + 0.9×0 + 0.9²×0 + 0.9³×100
= 72.9
Going left from State 4
γ = 0.5 (heavy)
0 + 0.5×0 + 0.5²×0 + 0.5³×100
= 12.5
Same path, discounted more
Financial analogy: γ is like an interest rate. A dollar today is worth more than a dollar in the future because you can invest it. For negative rewards (debts), discounting means it's better to delay paying — the RL algorithm will naturally push negative rewards as far into the future as possible.

📊 Returns Under Different Policies (γ = 0.5)

The return depends on the actions you take (your policy). Here are three example policies:

S1 (terminal) 100 100 100

S3 25 2.5 25

S5 6.25 20 20

  • Always go right — best from S5 (only 1 step to the +40 terminal), but poor for all other states.
  • Mixed policy (left everywhere, right from S5) — gets the best of both: S5 returns 20, all others match "always left". This is the optimal policy for this setup.
Worked example (S5 → right, γ=0.5):
Return = R(S5) + γ·R(S6) = 0 + 0.5 × 40 = 20

📉 Negative Rewards & Discounting

  • All rewards in the rover example were ≥ 0, but real environments often have negative rewards (penalties).
  • When rewards are negative, the discount factor incentivises the agent to delay negative rewards as long as possible.
  • Financial analogy: If you owe someone $10, it's better to pay later (future dollars are worth less due to interest).
  • This behaviour — avoiding or deferring negative outcomes — is often exactly the right thing for the system to do.

✅ Summary — The Return

  • The Return = weighted sum of all rewards along a trajectory, with discount factor γ applied at each step.
  • γ < 1 makes the algorithm prefer sooner rewards; γ close to 1 means future rewards are almost as valuable as immediate ones.
  • Different action sequences (policies) produce different returns — the goal of RL is to find the policy that maximises the return.
  • For negative rewards, discounting naturally encourages deferring them as far into the future as possible.

Lecture 4 — Policies

🗺️ What is a Policy?

  • A policy π is a function that maps every state s to an action a: π(s) = a.
  • It is the "brain" of the agent — given any situation, it tells the agent exactly what to do.
  • Also called a controller in some communities; "policy" is the standard RL term.
  • Goal of RL: find the policy π that maximises the return from every state.
In one sentence: A policy is a complete decision rule — it doesn't just say what to do now, it specifies an action for every possible state the agent could ever be in.

Many different policies are possible. Here are four, listed from naive to optimal:

  • Every row in the table above is a valid policy. RL's job is to discover the best one automatically.

Lecture 5 — MDP Formalism & Review

🔁 The Same Formalism Applies Everywhere

The (State, Action, Reward, Discount, Return, Policy) framework is universal. Here's how it maps to three different problems:

Component 🔴 Mars Rover 🚁 Helicopter ♟️ Chess
States 6 positions (1–6) Position, orientation, speed All piece positions on board
Actions Go left / Go right How to move the control sticks All legal moves
Rewards +100 (S1), +40 (S6), 0 (others) +1 flying well; −1,000 on crash +1 win, −1 loss, 0 tie
γ 0.5 (illustrative) ≈ 0.99 0.99 – 0.999
Policy π(s) Which direction to move Control stick movements given position Best move given board state
Chess uses γ very close to 1 (e.g. 0.999) because the only meaningful reward comes at the very end of the game — heavy discounting would make winning feel too distant to optimise for.

📐 Markov Decision Process (MDP)

  • The formal name for the RL framework we've been building is a Markov Decision Process (MDP).
  • It's not a new idea — it's just the official name for the (s, a, R, γ, π) formalism.
The Markov Property
The future depends only on the current state, not on the history of how you got there.
In other words: "It doesn't matter how you got here — only where you are now determines what happens next."
  • This property simplifies decision-making enormously: the agent only needs to observe the current state s — no memory of past states is required.
  • For the rover: being in State 4 is the same regardless of whether you came from State 3 or State 5.

🔃 The Agent–Environment Interaction Loop

  • The MDP can be visualised as a continuous loop between the agent and the environment:
Agent
Chooses action a
using policy π(s)
⇄
Environment
Returns new state s'
and reward R(s)
Agent observes: s, R(s) Agent outputs: a = π(s) World transitions to: s'
  • This loop repeats at every time step until a terminal state is reached.
  • The agent's only lever is its choice of action — the environment's dynamics (how states transition) are fixed.

⏭️ What's Next — The State-Action Value Function

  • We now have a complete description of the RL problem via the MDP formalism.
  • The next challenge: how does the algorithm actually find a good policy?
  • The key tool is the state-action value function (also called the Q-function): for every (state, action) pair, it estimates the return you'd get by taking that action and then following the best possible policy afterwards.
  • Learning to compute Q will be the foundation for building a practical RL algorithm.

Lecture 6 — The Q-Function

🎯 The State-Action Value Function Q(s, a)

  • Also called the Q-function (or Q* / optimal Q-function in the literature).
  • It is a function of a state s and an action a.
  • Q(s, a) = the return you get if you:
    1. Start in state s.
    2. Take action a once.
    3. Then behave optimally for all remaining steps.
  • It places no judgment on whether action a is wise — it faithfully reports the return for that choice followed by optimal play.
Note on circularity: The definition assumes we already know optimal behaviour, which seems circular. In practice, RL algorithms resolve this — Q can be computed iteratively even before the optimal policy is known. This is addressed in later lectures.

💡 How Q(s, a) Gives Us the Optimal Policy

  • The best possible return from state s is simply the largest Q-value over all actions:
maxa Q(s, a)
  • The optimal action to take in state s is the action that achieves this maximum:
π(s) = argmaxa Q(s, a)
  • Example — State 4: Q(S4, ←) = 12.5 and Q(S4, →) = 10. The max is 12.5, so the optimal action is ← Left.
  • Example — State 5: Q(S5, ←) = 6.25 and Q(S5, →) = 20. The max is 20, so the optimal action is → Right.
  • This is the payoff of computing Q: once you have it for all (s, a) pairs, reading off the optimal policy is trivial — just pick the action with the highest Q-value at each state.
Why Q is powerful: Q(s, a) answers "how good is it to take action a right now and then play perfectly?" — so maximising over a naturally surfaces the best first move, which is the optimal policy.

✅ Summary — The Q-Function

  • Q(s, a) = expected return starting from state s, taking action a once, then acting optimally.
  • Also written Q* or called the optimal Q-function in the literature — all the same thing.
  • The optimal policy follows directly: π(s) = argmaxa Q(s, a).
  • The best possible return from any state = maxa Q(s, a).
  • The key challenge — and the subject of upcoming lectures — is computing Q(s, a) efficiently without already knowing the optimal policy.

Lecture 7 — How Q(s, a) Changes with Rewards & γ

🧪 How Q(s, a) Responds to Parameter Changes

  • Q(s, a) and the optimal policy are not fixed — they shift whenever you change the reward function or the discount factor γ.
  • Two levers to experiment with on the Mars Rover:
    • Terminal rewards (left = 100, right = 40 by default)
    • Discount factor γ (default 0.5 in examples)
  • The optimal policy is always: pick the action with the larger Q-value at each state.

🔽 Experiment 1 — Reduce Right Terminal Reward to 10

  • Setup: terminal-left = 100, terminal-right = 10, γ = 0.5.
  • From State 5 (one step from the right terminal):
    • Q(S5, ←Left) = 6.25 — go left, eventually reach +100.
    • Q(S5, →Right) = 5 — go right, reach +10 immediately.
  • Even though State 5 is one step from the right terminal, the reward is now so small that going all the way left is better.
  • Optimal policy: go ← left from every state (including S5).
Insight: Proximity alone doesn't determine the optimal action — the magnitude of the reward matters too. A nearby small reward can be beaten by a distant large one.

⏫ Experiment 2 — Increase γ to 0.9 (more patient)

  • Setup: terminal-left = 100, terminal-right = 40, γ = 0.9.
  • A higher γ means future rewards are discounted less — the agent is more patient and willing to travel further for a bigger reward.
  • From State 5:
    • Q(S5, ←Left) ≈ 65.61
    • Q(S5, →Right) = 0.9 × 40 = 36
  • Optimal policy: go ← left from every state, including S5 — the patient agent prefers the much larger +100 reward even from nearby.
Check: Q(S5, →Right) = γ × 40 = 0.9 × 40 = 36. The one-step discount is only ×0.9, so the right reward barely shrinks — yet the left reward at +100 still dominates.

⏬ Experiment 3 — Decrease γ to 0.3 (very impatient)

  • Setup: terminal-left = 100, terminal-right = 40, γ = 0.3.
  • A very low γ heavily penalises distant rewards — the agent becomes extremely impatient.
  • From State 4: the agent no longer has the patience to travel to +100. The closer +40 reward wins despite being smaller.
  • Optimal policy changes: from State 4 (and beyond), go → right toward the +40 terminal instead of left toward +100.
Insight: With γ = 0.3, reaching +100 requires three discounting steps (0.3³ = 0.027), so the effective value of +100 from State 4 is tiny. The +40 nearby becomes the rational choice.

📋 Summary — What Drives Q and the Optimal Policy

Change Effect on agent Policy shift at S5
Right reward 40 → 10 (γ=0.5) Smaller nearby reward no longer worth it ← Left (was already ← but now even more so)
γ 0.5 → 0.9 More patient; future rewards barely discounted ← Left (ignores close +40)
γ 0.5 → 0.3 Very impatient; distant rewards heavily penalised → Right from S4+ (settles for nearby +40)
  • Both the reward magnitudes and γ jointly determine Q(s, a) and the optimal policy.
  • The optimal return from any state is always maxa Q(s, a) — the larger of the two Q-values.

Lecture 8 — The Bellman Equation

🔔 The Bellman Equation

  • The key equation in RL for computing Q(s, a).
  • Uses the following notation:
    • s — current state
    • a — current action
    • R(s) — reward received at current state (also called the immediate reward)
    • s' — next state after taking action a
    • a' — action taken in the next state s'
Bellman Equation
Q(s, a) = R(s) + γ · maxa' Q(s', a')
  • R(s) — the immediate reward for being in state s.
  • γ · maxa' Q(s', a') — the discounted best possible return from the next state s'.
  • Terminal states: there is no s', so the second term disappears: Q(s, a) = R(s) only.

✏️ Bellman Equation — Worked Examples (γ = 0.5)

Q(S2, →Right)
  • s = S2, a = →Right → s' = S3
  • R(S2) = 0
  • Q-values at S3: Q(S3, ←) = 25, Q(S3, →) = 6.25 → max = 25
Q(S2, →) = 0 + 0.5 × 25 = 12.5
Q(S4, ←Left)
  • s = S4, a = ←Left → s' = S3
  • R(S4) = 0
  • Q-values at S3: Q(S3, ←) = 25, Q(S3, →) = 6.25 → max = 25
Q(S4, ←) = 0 + 0.5 × 25 = 12.5
Terminal States (S1, S6)
  • No next state s' exists → second term drops out.
Q(S1, any) = R(S1) = 100
Q(S6, any) = R(S6) = 40

🧩 Intuition — Two-Part Decomposition

  • The total return from state s can always be split into exactly two parts:
Part 1 — Immediate reward
R(s)
R₁ in the reward sequence. You collect this reward simply by being in state s.
Part 2 — Discounted future return
γ · maxa' Q(s', a')
The best return achievable from the next state s', scaled by γ.
  • The full return sequence is: R₁ + γR₂ + γ²R₃ + … = R₁ + γ(R₂ + γR₃ + …)
  • The bracketed part (R₂ + γR₃ + …) is exactly the optimal return starting from s'. This is maxa' Q(s', a').
  • Hence: Q(s,a) = R(s) + γ · maxa' Q(s', a') — the Bellman equation.
Concrete breakdown — Q(S4, ←Left)
Full return: 0 + 0.5×0 + 0.5²×0 + 0.5³×100 = 12.5
Split:      R(S4) + 0.5 × (optimal return from S3)
            = 0 + 0.5 × 25 = 12.5 ✓

✅ Summary — The Bellman Equation

  • The Bellman equation gives a recursive way to compute Q(s, a) for any state-action pair.
  • Every Q-value equals the immediate reward plus the discounted best future return.
  • At terminal states: Q(s, a) = R(s) — no future term.
  • Computing Q via Bellman will be the foundation of the RL algorithm built in subsequent lectures.

Lecture 9 (Optional) — Stochastic Environments

🎲 Stochastic Environments

  • In practice, actions don't always produce the intended outcome — wind, slippery floors, wheel slip, and rocks can all cause the robot to end up somewhere unexpected.
  • A stochastic environment (or stochastic MDP) models this: taking action a in state s leads to a random next state s'.

Mars Rover example — command: go ←Left

90% of the time
Successfully goes left → ends up in S3
10% of the time
Slips and goes right → ends up in S5
  • The same holds in reverse: command →Right has a 90% chance of reaching S5 and a 10% chance of reaching S3.

📐 The Goal: Maximise Expected Return

  • Because the return is now a random number, we can't simply maximise it — it changes every episode.
  • Instead, the goal becomes maximising the expected return: the average sum of discounted rewards over many episodes.
  • In statistics, "expected value" = average. The notation is:
Maximise 𝔼[ R₁ + γR₂ + γ²R₃ + … ]
𝔼 = expected value (average over all possible random trajectories)
  • The policy π(s) is chosen to maximise this expectation.
  • Effect of stochasticity on Q-values: because the rover sometimes goes the wrong way, Q-values decrease — you have less control, so the best achievable return is lower.
  • Higher misstep probability → lower Q-values and lower optimal expected return.

🔔 Bellman Equation — Stochastic Version

  • The structure is the same, but the next state s' is now random, so we take an expectation over it:
Stochastic Bellman Equation
Q(s, a) = R(s) + γ · 𝔼[ maxa' Q(s', a') ]
DeterministicStochastic
Next state s' Fixed — determined by (s, a) Random — drawn from a distribution
Future return term γ · maxa' Q(s', a') γ · 𝔼[ maxa' Q(s', a') ]
Optimisation goal Maximise return Maximise expected return

✅ Summary — Stochastic MDPs

  • A stochastic MDP models real-world uncertainty: actions may not produce the commanded result.
  • The return is now a random variable — the goal shifts to maximising the expected (average) discounted return.
  • The Bellman equation gains an expectation operator 𝔼 over the random next state s'.
  • Higher randomness → lower Q-values, because reduced control means the best reachable return is lower.
