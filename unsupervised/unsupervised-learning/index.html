🔵 What is Clustering?

  • A clustering algorithm looks at a number of data points and automatically finds data points that are related or similar to each other.
  • It is an unsupervised learning algorithm — the dataset contains only inputs x, with no labels y.
  • Because there are no target labels, we cannot tell the algorithm what the "right answer" is — instead, we ask it to find interesting structure in the data.
  • Clustering looks for one specific type of structure: groups of points (clusters) that are similar to each other.

⚖️ Supervised vs. Unsupervised Learning

Supervised Learning Unsupervised Learning
Training data Input features x and labels y Input features x only — no labels
Plot appearance Two classes shown with distinct markers (× and ○) Just dots — no class distinction visible
Algorithm goal Learn a decision boundary (e.g. logistic regression, neural network) Discover structure or clusters in the data

🌐 Applications of Clustering

  • News article grouping — grouping similar news stories together (e.g. all articles about pandas).
  • Market segmentation — at deeplearning.ai, clustering revealed distinct learner groups: those wanting to grow skills, advance careers, or stay updated with AI in their field.
  • DNA data analysis — grouping individuals by genetic expression data to find people with similar traits.
  • Astronomy — grouping bodies in space to identify galaxies or other coherent structures; one of the most fascinating real-world uses of clustering.
Next up: The most commonly used clustering algorithm — the K-means algorithm.

📍 K-Means Clustering — Overview

  • Given an unlabelled dataset (e.g. 30 points), K-means tries to find a specified number of clusters K.
  • The algorithm repeatedly performs two steps until convergence:
    1. Assign every point to its nearest cluster centroid.
    2. Move each cluster centroid to the mean (average position) of all points assigned to it.
  • The centers of the clusters are called cluster centroids.
Key idea: K-means alternates between assigning points to centroids and moving centroids to the mean of their assigned points — repeating until nothing changes.

🔄 K-Means: Step-by-Step Walkthrough

  • Initialisation: Randomly place K cluster centroids (e.g. a red cross and a blue cross). These are just starting guesses — not necessarily good ones.
  • Step 1 — Assign points: For each data point, check which centroid it is closest to and assign it to that cluster (illustrated by colouring the point red or blue).
  • Step 2 — Move centroids: Look at all red points and compute their average location; move the red centroid there. Do the same for all blue points.
  • Repeat: With the centroids in new positions, reassign all points again. Some points may switch colour because the centroids have moved.
  • Convergence: Keep repeating Steps 1 and 2 until no point changes its cluster assignment and no centroid moves. At that point, K-means has converged.
Convergence condition: The algorithm has converged when applying both steps results in no further changes to point assignments or centroid locations.

📐 K-Means: Formal Algorithm

  • Initialisation: Randomly pick K cluster centroids μ1, μ2, …, μK. Each μk is a vector with the same dimension n as the training examples.
  • Step 1 — Assign points: For each training example x(i) (i = 1 … m), set c(i) = index k (1 … K) of the closest centroid:

    c(i) = argmink ‖x(i) − μk‖²

    Minimising the squared distance (L2 norm squared) is equivalent to minimising the distance and is more convenient to implement.
  • Step 2 — Move centroids: For each cluster k, update μk to the mean of all training examples assigned to it:

    μk = average of all x(i) where c(i) = k

    e.g. if cluster 1 has examples 1, 5, 6, 10 assigned to it: μ1 = (x(1) + x(5) + x(6) + x(10)) / 4
  • Repeat Steps 1 and 2 until convergence (no assignments or centroid positions change).
Note: μk always has the same dimensionality as the training data — if each x has n features, so does each centroid.

⚠️ Corner Cases & Practical Notes

  • Empty cluster: If a cluster ends up with zero points assigned to it, computing its mean is undefined. Two options:
    • Most common: Eliminate that cluster — you end up with K − 1 clusters.
    • Alternative: Randomly re-initialise that centroid and try again.
  • Non-separated clusters: K-means works fine even when clusters are not visually distinct. Example: sizing t-shirts (S/M/L) from a continuous height/weight dataset — K-means still finds useful groupings even without clear gaps in the data.
  • The cluster centroids from such a run give you a sense of the most representative point in each group (e.g. the ideal body dimensions to design each t-shirt size around).

📉 Why Does K-Means Converge?

  • K-means is not just a heuristic loop — it is actually optimising a cost function at every step.
  • Each of the two steps either decreases the cost or keeps it the same — it can never increase it.
  • Because the cost is bounded below (it cannot go negative), the algorithm is guaranteed to converge and will not run forever.
Coming next: The specific cost function K-means optimises — the distortion function — and how understanding it helps us run K-means better.

💰 K-Means Cost Function — The Distortion

  • Like supervised learning algorithms, K-means optimises a cost function J — but it uses its own two-step algorithm rather than gradient descent.
  • Notation recap:
    • c(i) — index (1 … K) of the cluster to which training example x(i) is currently assigned.
    • μk — location (vector) of cluster centroid k.
    • μc(i) — the centroid of the cluster that example x(i) has been assigned to.
  • Cost function (distortion):

    J(c(1)…c(m), μ1…μK) = (1/m) Σ ‖x(i) − μc(i)‖²

    The average squared distance between every training example and the centroid it has been assigned to.
  • This cost function is also called the distortion function in the literature.
Visually: J is computed by measuring the distance from every point to its assigned centroid, squaring each distance, and averaging them all. A lower J means tighter, better-fit clusters.

🧮 Why Each Step Minimises J

  • Step 1 — Assign points (update c(i)): Holding the centroids μ fixed, assigning each x(i) to the nearest centroid minimises the term ‖x(i) − μc(i)‖² for that point. Any other assignment would only make the distance larger.
  • Step 2 — Move centroids (update μk): Holding assignments c(i) fixed, the value of μk that minimises the average squared distance to all assigned points is exactly their mean.
    • Example: two points at positions 1 and 11. Centroid at 1 → avg sq. distance = 41. Centroid at mean (6) → avg sq. distance = 25. The mean is always optimal.
Key insight: Every single iteration of K-means either decreases J or keeps it the same — it can never increase. This is why convergence is guaranteed.

🛑 Convergence & Debugging Tips

  • If J ever goes up on an iteration, there is a bug in the code — this should be mathematically impossible.
  • If J stops decreasing, K-means has converged — you can safely stop.
  • If J is decreasing but very slowly, it is similar to gradient descent: you can choose to stop early if the rate of improvement is negligible.
  • Plotting J over iterations is a practical way to monitor whether the algorithm is working correctly and to detect convergence.
Coming next: Using multiple random initialisations of cluster centroids to find better clusters and avoid poor local minima.

🔢 Choosing the Number of Clusters K

  • K is a required input to K-means, but for many datasets the right value is truly ambiguous — different people looking at the same data may reasonably see 2 or 4 clusters, and both could be correct.
  • Because there are no labels in unsupervised learning, there is no ground truth to compare against.
  • What does NOT work: Choosing K to minimise J — more clusters always reduces J, so this approach would almost always push you toward the largest possible K.

💪 The Elbow Method

  • Run K-means for a range of K values and plot the distortion J against K.
  • If the curve has a clear elbow — a point where J drops sharply then levels off — choose that K (e.g. K = 3 if the rapid decrease stops there).
  • Limitation: In practice, many cost-function curves decrease smoothly with no clear elbow, making the method unreliable. It is mentioned here because you will encounter it in the literature, but it is not widely recommended.

🎯 Choosing K via Downstream Purpose (Recommended)

  • The most practical approach: evaluate K-means based on how well its output serves the later downstream task it feeds into.
  • T-shirt sizing example:
    • K = 3 → S, M, L — simpler, cheaper to manufacture and ship.
    • K = 5 → XS, S, M, L, XL — better fit for customers, but higher production costs.
    • Both are valid clusterings; the business trade-off (fit quality vs. cost) determines the right K.
  • Image compression example: K-means can compress images by reducing the number of colours. Higher K → better image quality but larger file size. You choose K based on the acceptable trade-off between quality and compression.
Recommended approach: Run K-means with a few values of K, then pick the one that makes the most sense in the context of what you will actually do with the clusters.

🚨 Up Next — Anomaly Detection

  • The second unsupervised learning algorithm covered in this course.
  • Goal: look at a dataset and automatically identify unusual or anomalous data points.
  • One of the most commercially important applications of unsupervised learning, used across many industries.

🚨 Anomaly Detection — Introduction

  • An anomaly detection algorithm looks at an unlabelled dataset of normal events and learns to raise a red flag when an unusual or anomalous event occurs.
  • Motivating example — aircraft engines:
    • After an engine rolls off the assembly line, features are computed: x1 = heat generated, x2 = vibration intensity.
    • Most manufactured engines are fine, so the dataset is mostly normal examples.
    • Given a new engine xtest, the question is: does it look similar to engines seen before, or is something unusual about it?

📊 Density Estimation

  • The most common approach to anomaly detection is density estimation.
  • Given a training set of m examples, the algorithm builds a model for p(x) — the probability of seeing a particular set of feature values.
  • Regions dense with training examples get high probability; sparse outer regions get low probability.
  • Decision rule:
    • If p(xtest) < ε → flag as anomaly (the point lands in a low-probability region).
    • If p(xtest) ≥ ε → okay, looks similar to normal examples.
  • ε (epsilon) is a small threshold you choose — think of it as "how unlikely is too unlikely."
Key idea: An anomaly is not just a data point far from the centre — it is a data point in a region the model considers very unlikely given the training data.

🌐 Applications of Anomaly Detection

Domain Features (examples) What it detects
Fraud detection Login frequency, pages visited, transaction count, typing speed Unusual user behaviour → trigger extra security checks (e.g. phone verification)
Manufacturing Heat, vibration, electrical properties of parts Defective engines, circuit boards, smartphones, motors before they ship
Data centre monitoring Memory usage, disk accesses/sec, CPU load, CPU-to-network-traffic ratio Failing hard drives, network cards, or compromised machines
Telecommunications Cell tower behaviour metrics Towers behaving unusually → dispatch a technician
Financial fraud Purchase patterns, transaction amounts Unusual sequences of purchases worth flagging for review
Note: Anomaly detection typically does not automatically block or close an account. Instead, it flags the case for a human or a secondary security step to review.

🔔 The Gaussian (Normal) Distribution

  • Gaussian distribution, normal distribution, and bell-shaped curve all refer to the same thing.
  • If x is a random variable distributed as a Gaussian with mean μ and variance σ², then its probability is given by:

    p(x) = (1 / √(2π) · σ) · exp(−(x − μ)² / 2σ²)

  • Parameters:
    • μ (mu) — the mean; controls the centre of the curve.
    • σ (sigma) — the standard deviation; controls the width of the curve.
    • σ² (sigma squared) — the variance.
  • The area under the curve always equals 1 (probabilities must sum to one), so a narrower/taller curve and a wider/shorter curve are equivalent in total area.

📈 Effect of μ and σ on the Gaussian

μ σ Shape
0 1 Centred at 0, standard width — the standard normal distribution.
0 0.5 Centred at 0, narrower and taller (σ² = 0.25).
0 2 Centred at 0, wider and shorter (σ² = 4).
3 0.5 Shifted right to centre at 3, same width as σ = 0.5 case above.
Rule of thumb: μ moves the curve left/right; σ stretches or squishes it. A wider curve must be shorter to keep the total area = 1.

🧮 Estimating μ and σ² from Data

  • Given a training set of m scalar examples, estimate the parameters as follows:

    μ = (1/m) Σ x(i)

    σ² = (1/m) Σ (x(i) − μ)²

  • These are the maximum likelihood estimates for μ and σ.
  • Some statistics texts use 1/(m−1) instead of 1/m for σ² — in practice the difference is negligible. Using 1/m is standard in ML.
  • Once μ and σ² are estimated, you have a fitted Gaussian that describes how likely any value of x is given the training data.
Anomaly detection link: A point near the centre of the fitted Gaussian has high p(x) → normal. A point far from the centre has low p(x) → potential anomaly.

🔢 Anomaly Detection with Multiple Features

  • Training set: x(1) … x(m), where each example x is a vector of n features.
  • Model p(x) as the product of individual feature probabilities:

    p(x) = p(x1; μ1, σ1²) × p(x2; μ2, σ2²) × … × p(xn; μn, σn²)

    Or more compactly: p(x) = ∏j=1n p(xj; μj, σj²)
  • This assumes the features are statistically independent, though the algorithm works well in practice even when they are not.
  • Why multiply probabilities? If there is a 1/10 chance an engine runs very hot, and a 1/20 chance it vibrates very hard, the chance of both happening is 1/10 × 1/20 = 1/200 — very unlikely, and therefore anomalous.

🛠️ Building the Model — Step by Step

  1. Choose features xj that might be indicative of anomalous behaviour.
  2. Fit parameters for each feature j = 1 … n from the training set:

    μj = (1/m) Σ xj(i)

    σj² = (1/m) Σ (xj(i) − μj)²

    Vectorised: μ = (1/m) Σ x(i) (computes all μj at once).
  3. Given a new example x, compute p(x):

    p(x) = ∏j=1n (1 / √(2π) σj) · exp(−(xj − μj)² / 2σj²)
  4. Flag as anomaly if p(x) < ε.
Intuition: The algorithm flags an example as anomalous if one or more features are unusually large or small. Even a single very unlikely feature value makes the whole product p(x) very small.

✏️ Worked Example (2 Features)

  • Dataset with features x1 (wide range) and x2 (narrow range). Fitted parameters:
    • μ1 = 5, σ1 = 2
    • μ2 = 3, σ2 = 1
  • Multiplying p(x1) × p(x2) produces a 3-D surface — highest in the centre of the training data, falling off toward the edges.
  • With ε = 0.02:
    • xtest 1 (near the centre): p(x) ≈ 0.4 → 0.4 ≥ ε → OK, not an anomaly.
    • xtest 2 (x1 ≈ 8, x2 ≈ 0.5, far from training data): p(x) ≈ 0.0021 → 0.0021 < ε → flagged as anomaly.
Coming next: How to choose ε and evaluate whether your anomaly detection system is working well.

📋 Evaluating an Anomaly Detection System

  • Making decisions about features or ε is much easier if you have a real number evaluation — a single metric that tells you if a change made the algorithm better or worse.
  • Even though anomaly detection is unsupervised, a small number of labelled anomalies (y = 1) are very useful for evaluation. Normal examples get y = 0.
  • The training set remains unlabelled — all examples are assumed normal (y = 0). The algorithm still fits Gaussian distributions on this set.
  • A small number of known anomalies are reserved for the cross-validation and test sets, which contain both y = 0 and y = 1 examples.

✂️ Dataset Splitting

  • Typical split (aircraft engine example): 10,000 good engines + 20 known anomalies.
    • Training set: 6,000 good engines (y = 0 only).
    • Cross-validation set: 2,000 good + 10 anomalous engines.
    • Test set: 2,000 good + 10 anomalous engines.
  • Alternative — no test set: If anomalies are very few (e.g. only 2), put all anomalies into the cross-validation set and skip the test set entirely.
    • Downside: no fair way to estimate real-world performance; higher risk of over-fitting ε and feature choices to the CV set.
  • A few anomalies accidentally included in the training set will not significantly harm the algorithm.

📐 Evaluation Procedure & Metrics

  • Procedure:
    1. Fit model p(x) on the training set (fit all Gaussians).
    2. On each CV / test example, predict y = 1 (anomaly) if p(x) < ε, else y = 0.
    3. Compare predictions to the known labels y in the CV / test set.
  • Metrics: Because anomalies are rare (heavily skewed classes — e.g. 10 positives vs. 2,000 negatives), plain accuracy is misleading. Prefer:
    • True positive / false positive / false negative / true negative rates.
    • Precision, Recall, and F₁ score.
  • Use the CV set to tune ε: raise it to catch more anomalies; lower it to reduce false alarms on normal examples.

⚖️ Anomaly Detection vs. Supervised Learning

Anomaly Detection Supervised Learning
Positive examples (y=1) Very few — 0 to ~20 is common Enough to give the algorithm a sense of what positives look like
Negative examples (y=0) Large set — used to model p(x) Large set used in training alongside positives
What it learns What normal looks like; flags anything that deviates What positive examples look like; flags things similar to past positives
Future anomalies Can detect brand-new, never-seen types of anomaly Assumes future positives are similar to those in the training set
Where positives are used Only in CV and test sets for tuning ε Directly in training

🗺️ When to Use Each — Examples

  • Use anomaly detection when:
    • There are many different, unpredictable ways for something to go wrong.
    • New types of anomalies may appear in the future that were never seen during training.
    • You have very few labelled positive examples to learn from.
  • Use supervised learning when:
    • You have a reasonable number of positive examples to train on.
    • Future positives are expected to look similar to past ones.
    • The output labels repeat reliably over time.
Task Preferred Approach Reason
Financial fraud (new schemes) Anomaly detection New fraud types emerge constantly; can't rely on past positives
Email spam detection Supervised learning Spam is repetitive — future spam resembles past spam
Manufacturing — new/unseen defects Anomaly detection Brand-new failure modes may arise with no prior examples
Manufacturing — known defects (e.g. scratches) Supervised learning Enough labelled scratch examples to train a classifier directly
Data centre / security monitoring Anomaly detection Hackers constantly find new attack methods unlike anything seen before
Weather prediction Supervised learning Fixed set of output labels (sunny, rainy, snow) seen repeatedly
Disease diagnosis (known disease) Supervised learning Training on known symptom patterns and seen diagnoses
Key distinction: Anomaly detection flags things unlike anything normal. Supervised learning flags things like the positives it has already seen.

🔧 Feature Engineering for Anomaly Detection

  • In supervised learning, the labelled signal y helps the algorithm figure out which features matter and how to re-scale them — so imperfect features are more forgivable.
  • In anomaly detection (unsupervised), the algorithm has no such signal, so carefully choosing features is even more important.
  • Key goal: Give the algorithm features that take on unusually large or small values when something is anomalous.

🔔 Making Features More Gaussian

  • Anomaly detection models each feature with a Gaussian — so features that are already roughly Gaussian give the best fit.
  • Plot a histogram of each feature. If the distribution looks skewed or non-symmetric, apply a transformation to make it more bell-shaped before training.
  • Common transformations to try:
    • log(x) or log(x + c) — good for right-skewed data. If x has zeros, use a small constant c (e.g. 0.001) to avoid log(0).
    • x0.5 (square root), x0.4, or other fractional powers.
    • x1/3 (cube root).
    • More generally: xp — adjust the exponent p until the histogram looks symmetric.
  • Try a few values of the transformation parameter, plot the histogram each time, and pick the one that looks most Gaussian — no need to be mathematically precise.
Important: Whatever transformation you apply to the training set, apply the exact same transformation to your cross-validation and test sets.

🔍 Error Analysis for Anomaly Detection

  • After training, evaluate on the cross-validation set. If the algorithm misses anomalies, perform error analysis:
    1. Look at the anomalous examples that the algorithm failed to flag.
    2. Ask: what makes this example anomalous even though its current features look normal?
    3. Try to identify a new feature that takes on an unusually large or small value for that example.
    4. Add the new feature and retrain.
  • Target: p(x) should be large (≥ ε) for normal examples and small (< ε) for anomalous ones. The most common failure mode is p(x) being large for both.

⚙️ Creating New Features by Combining Old Ones

  • Sometimes an anomaly is not unusual on any single feature but is unusual on a combination of features.
  • Data-centre monitoring example:
    • Base features: memory use (x1), disk accesses/sec (x2), CPU load (x3), network traffic (x4).
    • A machine with high CPU load but very low network traffic is anomalous for a video-streaming server — but neither feature alone looks unusual.
    • Fix: add x5 = CPU load / network traffic. This new ratio spikes for exactly that failure mode and is easy for the algorithm to flag.
    • Can also try: x6 = (CPU load)² / network traffic or other combinations.
  • Fraud detection example: A user's transaction count (x1) may look normal, but adding typing speed (x2) — which is extreme for that user — makes the anomaly obvious in 2-D feature space.
Workflow: Train → find missed anomalies on CV set → inspect them → create new features → retrain → repeat.
